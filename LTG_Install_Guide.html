Install Guide · Windows & Mac

Everything installs into one folder. You'll do the one-time setup exactly once — after that, playing (or updating) is just double-clicking a launcher.

01One-Time Setup

Install two tools, then pull down the game itself. Follow the column for your machine.

a. Install Python

Windows
  1. Go to python.org/downloads and click the big yellow Download Python button.
  2. Run the installer. On the first screen, tick the box that says "Add python.exe to PATH" (bottom of the window) — this matters.
  3. Click Install Now and let it finish.
Mac
  1. Open Terminal (Spotlight → type Terminal → Enter).
  2. Type python3 --version and press Enter.
  3. If a version number prints, you already have Python — skip ahead. If a dialog offers to install Command Line Tools, click Install and wait for it to finish; this brings Python 3 with it.

b. Install Git

Windows
  1. Go to git-scm.com/download/win — the download starts automatically.
  2. Run the installer. Every default is fine — just keep clicking Next, then Install.
Mac
  1. In the same Terminal window, type git --version and press Enter.
  2. If it's not already installed, macOS will offer to install the Command Line Tools — same prompt as Python above. Click Install and wait.

c. Download the game

Windows
  1. Open File Explorer and go to the folder you want the game to live in (for example Documents).
  2. Click in the address bar at the top, type cmd, and press Enter — a black command window opens in that folder.
  3. Copy-paste this line into it and press Enter:
    git clone https://github.com/LocalCalzoneZone/LTG.git
    A folder called LTG appears with the whole game inside.
Mac
  1. In Terminal, navigate to where you want the game to live, for example: cd ~/Documents
  2. Run:
    git clone https://github.com/LocalCalzoneZone/LTG.git
    A folder called LTG appears with the whole game inside.

Don't Use "Download ZIP"

GitHub's green Code → Download ZIP button gives you a copy that can't receive updates. The git clone above is what makes the in-game Update button work later.

02Starting the Game

Inside the LTG folder:

GameLTG-Game.batLTG-Game.commandJust the game
DeckbuilderLTG-Deckbuilder.batLTG-Deckbuilder.commandJust the character / deck editor
  • The first launch sets everything up and takes a few minutes — that's normal. After that it's seconds.
  • A dark window opens and stays open — that window IS the app. Your browser then opens the game by itself. With LTG-Start the character editor runs quietly in the background — no extra tab; the game's Edit buttons reach it. Closing the window stops everything it started.
  • If the browser doesn't open on its own: the game is at localhost:8020, the deckbuilder at localhost:8000.
Windows

If Windows Firewall asks whether to allow it: click Allow.

Mac

The first time you double-click a .command file, macOS may warn it's from an unidentified developer. Right-click (or Control-click) the file → Open → Open again in the dialog. You only need to do this once per launcher.

Quitting: click Quit — it sits next to New Game in the game's top bar, and in the deckbuilder's top bar. A confirmation pops up first, because quitting is all-or-nothing: it shuts down BOTH the game and the deckbuilder servers (the terminal window closes too, and anyone connected to your game is disconnected), so nothing keeps running in the background. Anything you haven't saved is lost — Save first.

03Characters

Characters are .json files. They live only on your computer — updates never touch them — and they're small enough to send over any chat. This works the same on Windows and Mac.

Play a character someone sent you

  1. Save the .json file somewhere you can find it (e.g. Documents).
  2. Start LTG-Start → in the game: Options → Characters → Import → pick the file.
  3. It's now in your roster for any New Game.

Edit a character (the good path)

  1. Start with LTG-Start (it runs the game and the editor together).
  2. In the game: Options → Characters → find the character → Edit. The deckbuilder opens with that character loaded.
  3. Make your changes (cards, stats, portrait…).
  4. Click Update Game Character (top right). Done — the game uses the new version from the next New Game onward.

Send a character to someone

  1. Open the character in the Deckbuilder (via the Edit path above, or its Load button and picking a .json file).
  2. Click Save in the top bar and choose where to put the .json file.
  3. Send that file (email, chat, whatever). They import it with the Import button in their game.

04Updating the Game

Same steps on both platforms — when there's a new version:

  1. Start LTG-Start → in the game: Options → Settings → Updates.
  2. If it says updates are available, click Update now and wait — it can take a minute.
  3. Click Quit (next to New Game) and relaunch LTG-Start. That's it — one update covers the game, the deckbuilder, and any new encounters or adventures.

Your saved characters are never touched by an update.

05If Something Goes Wrong

Windows
  • The launcher window flashes an error about Python — Python probably isn't on PATH. Re-run the Python installer, tick Add python.exe to PATH, then delete the .venv folder inside LTG and double-click the launcher again.
  • Windows Firewall keeps re-asking — click Allow again; it only needs to happen once per launcher, but a Windows update can occasionally reset it.
Mac
  • "Cannot be opened because it is from an unidentified developer" — right-click the .command file → Open → Open again. See Starting the Game above.
  • Nothing happens when you double-click — open Terminal, cd into the LTG folder, and run ./LTG-Start.command directly to see the error text.

Applies to both platforms

  • An update said "delete the .venv folder" — do exactly that: close the app, delete the folder called .venv inside LTG, and double-click the launcher. It rebuilds itself.
  • The game page won't load — check that the terminal window is still open, then try localhost:8020 in your browser.
  • Anything else — screenshot the terminal window and send it to your game admin.
